# Contributing to the Sovereignty Certificate Specification

Thank you for your interest in contributing to the Sovereignty Certificate Specification. This document explains how to participate.

## How to Provide Feedback

### Reporting Issues

Use the [issue templates](https://github.com/Lucid-Computing/sovereignty-certificate-specification/issues/new/choose) to submit feedback. We provide templates for:

- **Editorial feedback** — Typos, grammatical errors, unclear wording, or formatting issues
- **Technical issues** — Protocol design concerns, security considerations, interoperability questions
- **General questions** — Questions about the specification or the working group

When filing an issue:

- Submit one issue per topic. Do not combine multiple concerns in a single issue.
- Reference specific section numbers (e.g., "Section 6.3.1" or "Annex A.2").
- Quote the relevant text when reporting editorial issues.

### Participating in Discussions

Use [GitHub Discussions](https://github.com/Lucid-Computing/sovereignty-certificate-specification/discussions) for:

- Open-ended conversations about the specification's direction
- Implementation experience reports
- Broader architectural questions
- Ideas for future versions

Discussions may result in formal issues being filed if they identify specific changes needed.

### Proposing Changes via Pull Request

1. Fork the repository and create a branch from `main`.
2. Make your changes to the specification file in `spec/`.
3. In the PR description, describe what you are changing and why.
4. Reference any related issue numbers.
5. Ensure section references within the document remain consistent.

Pull requests will be reviewed by the working group.

## Contribution Scope

This repository is for the specification text itself. Reference implementations, test suites, and tooling belong in separate repositories.
